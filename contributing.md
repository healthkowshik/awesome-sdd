# Contributing

Thank you for your interest in contributing to Awesome Spec-Driven Development!

## Adding a Resource

To suggest a new resource, please open a pull request with the following:

1. **Relevance**: The resource must be directly relevant to Spec-Driven Development (SDD) as a practice, methodology, or tooling category
2. **Quality**: The resource must originate from a credible source — established blogs, conference talks, recognized open-source projects, or peer-reviewed venues
3. **Formatting**: Use the entry format `- [Title](URL) - Description` where the description is a single sentence fragment with no trailing period
4. **Placement**: Add the entry to the correct category section (Articles, Tooling, or Videos)

## Quality Standards

- Tools must have a public repository or landing page with clear documentation
- Articles and videos must provide substantive content
- Links must resolve to live, accessible content
- Paywalled content must be clearly labeled

## Process

1. Fork the repository
2. Add your entry following the formatting guidelines above
3. Open a pull request targeting the `main` branch
4. An automated lint check will verify formatting compliance
5. A maintainer will review for quality, relevance, and formatting

## Running the Linter Locally

You can check your changes before submitting:

```bash
npx awesome-lint
```
